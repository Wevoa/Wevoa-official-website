About

Wevoa exists to make the full web stack feel smaller and more coherent.

The project combines a language, runtime, framework, package flow, and CLI because that lets the platform make stronger promises about how the pieces fit together.

Philosophy

Server-first by default

The normal path favors HTML, routes, and predictable request handling instead of pushing every app toward client-heavy complexity.

Identity

Framework and language together

Wevoa is not only a framework. It is a language, runtime, CLI, and web platform designed to feel coherent end to end.

Direction

Small surface, real capability

The goal is not to mimic every existing stack. The goal is to provide a simpler stack that still handles real production needs.

What makes Wevoa different

  • Language and framework are designed together
  • CLI, runtime, templates, routing, and build flow live in one system
  • HTML-first development is the normal path, not the fallback path
  • Windows installer and official distribution are part of the platform story

Current direction

Wevoa is already usable today for server-rendered products, dashboards, internal tools, and framework experimentation. The next stage is strengthening the ecosystem, security, docs, and adoption story around the core runtime.
