Ecosystem

A framework is stronger when the surrounding platform is coherent.

Wevoa's ecosystem is intentionally focused: official starter packs, official packages, installer-first distribution, and documentation that matches the actual runtime.

Packages

Official packages

Core packages such as @auth, @db, and @utils give projects a reliable starting point without making the framework feel bloated.

  • @auth for session-backed authentication
  • @db for database helpers and query wrappers
  • @utils for response and helper functions

Starters

Starter packs

Wevoa ships with focused built-in starters instead of a cluttered example zoo.

  • app for a minimal production-grade starting point
  • dashboard for an auth + CRUD starter built with official packages

Platform

Product ecosystem

The runtime, installer, packages, docs, and starters are designed to feel like one platform instead of separate tooling islands.

  • Single-binary runtime distribution
  • Installer-first onboarding on Windows
  • Package discovery and official registry flow

Official packages

@auth, @db, and @utils give projects a solid baseline without forcing a huge dependency tree.

Official starters

The built-in `app` and `dashboard` starters replace scattered demos with a cleaner onboarding path.
