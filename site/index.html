One language. One runtime. One server-first path.

Wevoa is the web stack when the language and framework are built as one system.

WevoaWeb combines a custom language, runtime, template engine, HTTP server, CLI, starter packs, and package system into one installable platform built in modern C++. The website should feel like that same idea: direct, technical, and unmistakably its own.

Capability map

What the platform already does well

Wevoa is already a working runtime and framework with routes, templates, package support, build flows, and production-minded behavior. The design language should communicate that confidence instead of hiding it behind generic SaaS patterns.

Execution

Language + runtime

Custom syntax, closures, imports, arrays, objects, template blocks, and route execution live in one binary.

Framework

Full web stack

Routing, sessions, CSRF, JSON helpers, multipart uploads, static assets, and SQLite are built into the platform.

CLI

Developer tooling

Project creation, doctor, build, serve, package management, and installer-ready distribution come from the same CLI.

Runtime

Production-oriented

Worker pool, bounded queue, runtime snapshots, compiled templates, and safer defaults are already part of the core.

Built with Wevoa

The official site is a Wevoa app too.

This website uses Wevoa routes, layouts, view rendering, and static file serving so the product can demonstrate itself through a real project instead of only screenshots and promises.

Request to HTML with one internal pipeline.
